CORE FEATURES - DETAILLIERT
================================================================================

1. AUDIO RECORDING HOOK (useAudioRecorder)
──────────────────────────────────────────

Hook-Struktur:
  const recorder = useAudioRecorder();

State-Variablen:
  - isRecording: boolean (Aufnahme läuft)
  - isPaused: boolean (Aufnahme pausiert)
  - duration: number (Sekunden)
  - audioBlob: Blob (Aufnahmen-Daten)

Methoden:
  - startRecording(): Promise
    → Fragt Microphone-Zugriff an
    → Initialisiert MediaRecorder
    → Startet Timer
    → Throws error bei Fehler

  - pauseRecording(): void
    → Pausiert MediaRecorder
    → Stoppt Timer

  - resumeRecording(): void
    → Resumed MediaRecorder
    → Startet Timer neu

  - stopRecording(): Promise<Blob>
    → Stoppt MediaRecorder
    → Erstellt Blob aus Chunks
    → Gibt Audio-Blob zurück

  - reset(): void
    → Setzt alle States zurück

Technische Details:
  - MediaRecorder API (Browser built-in)
  - MIME-Type: audio/webm;codecs=opus
  - Audio-Constraints: echoCancellation, noiseSuppression, autoGainControl
  - Speichert Audio in Chunks für Robustheit

2. SPEECH-TO-TEXT HOOK (useSpeechRecognition)
──────────────────────────────────────────────

Hook-Struktur:
  const speechAPI = useSpeechRecognition();

Methoden:
  - transcribeFromBlob(audioBlob): Promise<Array>
    → Nimmt Audio-Blob
    → Gibt Transkript-Array zurück
    → Format: [{ timestamp, speaker, text, edited }]

Fallback-Verhalten:
  - Web Speech API: Real-time während Aufnahme
  - Offline-Mode: Placeholder-Text mit Info
  - Empfehlung: OpenAI Whisper API für Produktion

Transkript-Format:
  [
    {
      timestamp: "00:00",      // MM:SS Format
      speaker: "System",        // oder Name
      text: "Gesprochener Text",
      edited: false            // Wurde Benutzer editiert?
    }
  ]

3. TOAST NOTIFICATION SYSTEM
─────────────────────────────

Toast-Komponente Props:
  - message: string (Text-Nachricht)
  - type: 'success' | 'error' | 'warning' | 'info'
  - onClose: callback

Verwendung:
  showToast('Aufnahme gestartet', 'success');
  showToast('Fehler: Microphone nicht gefunden', 'error');

Features:
  - Auto-Dismiss nach 4 Sekunden
  - Slide-in Animation von rechts
  - Schließen-Button
  - Position: Bottom-Right mit 24px Margin
  - z-index: 10000 (über alles)
  - Mobile-responsive

4. RECORDING PERSISTIERUNG
──────────────────────────

localStorage-Struktur:
  localStorage.setItem('smartmeets_recordings', JSON.stringify(recordings))

Recording-Objekt:
  {
    id: number (timestamp),
    title: string,
    date: string (lokalisiert),
    duration: number (sekunden),
    category: string,
    tags: array,
    transcriptLength: number
  }

Auto-Save Verhalten:
  - Triggert useEffect bei Änderungen
  - Nur wenn recordings.length > 0
  - JSON-serialisiert alle Daten
  - Load on Mount

5. EXPORT-FUNKTIONEN
────────────────────

TXT-Export:
  Format:
    ```
    Titel
    Datum
    Kategorie
    Tags
    
    ---
    
    [00:00] Speaker: Text
    [00:15] Speaker: Weiterer Text
    ```
  
  Download-Verhalten:
    - Erstellt Blob mit type 'text/plain'
    - Filename: "{Titel}_{timestamp}.txt"
    - Nutzt objektURL + Anchor-Link

PDF-Export:
  Format: Basisches PDF 1.4 (ohne externe Library)
  
  Inhalt:
    - Titel
    - Datum
    - Metadata
  
  Erweiterbar:
    - Braucht externe Library (jsPDF/PDFKit)
    - Oder Backend-API für Pro-Version

Copy-to-Clipboard:
  - Formatiert Transkript
  - Kopiert via navigator.clipboard.writeText()
  - Zeigt Success-Toast

STATE-MANAGEMENT
================================================================================

Top-Level State (SmartMeets Component):

Recording-Related:
  - isRecording: recorder.isRecording (aus Hook)
  - duration: recorder.duration (aus Hook)
  - audioBlob: recorder.audioBlob (aus Hook)
  - recordings: [] (alle bisherigen)

Content-Related:
  - transcript: [] (aktuelles Transkript)
  - uploadedFile: null (hochgeladene Datei)
  - meetingTitle: string
  - meetingCategory: string
  - meetingTags: []
  - newTag: string (Input-feld)

UI-State:
  - activeTab: 'record' | 'transcript' | 'history'
  - toast: { message, type, id } | null
  - isMobileMenuOpen: boolean
  - isTranscribing: boolean (Loading-State)

Flow:
  User startet Aufnahme
    ↓
  startRecording() → recorder.startRecording()
    ↓
  isRecording = true, Timer läuft
    ↓
  Benutzer klickt "Stoppen"
    ↓
  stopRecording() → blob erstellt
    ↓
  transcribeFromBlob(blob) → Transkript
    ↓
  Neue Recording gespeichert zu localStorage
    ↓
  activeTab wechselt zu 'transcript'
    ↓
  Toast: "Aufnahme erfolgreich"

FEHLERBEHANDLUNG
================================================================================

Fehlerquellen & Behandlung:

1. Microphone-Zugriff
   Fehler: navigator.mediaDevices.getUserMedia() rejected
   Handler: try-catch mit benutzer-freundlicher Message
   Toast: "Fehler: Microphone access denied"

2. Transkription
   Fehler: Speech API nicht verfügbar
   Handler: Fallback zu Placeholder-Text
   Toast: "Fehler bei Transkription"

3. File Upload
   Fehler: Ungültiges Format, zu groß
   Handler: Validation vor Upload
   Toast: "Upload-Fehler: ..."

4. localStorage
   Fehler: Quota exceeded, Disabled
   Handler: try-catch beim Laden
   Fallback: In-Memory State

5. Export
   Fehler: Blob-Erstellung fehlgeschlagen
   Handler: try-catch mit Error-Toast
   Fallback: Copy-to-Clipboard

================================================================================
DEBUGGING & TROUBLESHOOTING
================================================================================

1. Aufnahme funktioniert nicht
   - Check: Browser-Console für Errors
   - Check: Microphone-Zugriff erlaubt?
   - Check: HTTPS-Verbindung (required für getUserMedia)
   - Debug: console.log(recorder.isRecording)

2. Transkription leer
   - Normal: Web Speech API braucht Online-Modus
   - Fallback: Zeigt Placeholder-Text
   - Fix: Nutze OpenAI Whisper API
   - Debug: console.log(transcript)

3. Export funktioniert nicht
   - Check: popup-blocker
   - Check: Browser-Konsole Errors
   - Debug: console.log(blob)
   - Fallback: Copy-to-Clipboard

4. Daten nach Reload weg
   - Check: localStorage enabled?
   - Debug: window.localStorage in Console
   - Fix: Check browser-settings
   - Test: console.log(localStorage.getItem('smartmeets_recordings'))

5. Performance-Issues
   - Check: Große Audio-Dateien?
   - Check: Zu viele Recordings im localStorage?
   - Fix: Alte Aufnahmen löschen
   - Monitoring: Chrome DevTools Performance Tab
