🚀 QUICK START (5 MINUTES)
================================================================================

1. BACKEND SETUP
──────────────────

# Clone/copy backend folder
cd smartmeets-v2-backend

# Install dependencies
npm install

# Create .env file
cp .env.example .env

# Edit .env and add:
# - OPENAI_API_KEY=sk-xxx
# - DATABASE_URL=postgresql://...

# Start PostgreSQL (Docker)
docker run --name smartmeets-db \
  -e POSTGRES_DB=smartmeets \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=secure_password \
  -p 5432:5432 \
  -d postgres:15

# Initialize database
npm run db:init

# Start backend server
npm run dev

# ✅ Server should run on http://localhost:3000


2. FRONTEND SETUP
──────────────────

# Clone/copy frontend folder
cd smartmeets-v2-frontend

# Install dependencies
npm install

# Create .env file
cat > .env.local << EOF
VITE_API_URL=http://localhost:3000/api
EOF

# Start development server
npm run dev

# ✅ App should run on http://localhost:5173

================================================================================
📚 API ENDPOINTS
================================================================================

Authentication:
────────────────

POST /api/auth/register
  Body: { email, name, password }
  Response: { token, user }

POST /api/auth/login
  Body: { email, password }
  Response: { token, user }

GET /api/auth/profile
  Headers: Authorization: Bearer {token}
  Response: { user }

PUT /api/auth/profile
  Headers: Authorization: Bearer {token}
  Body: { name?, avatarUrl?, timezone?, language? }
  Response: { success, message }

POST /api/auth/logout
  Headers: Authorization: Bearer {token}
  Response: { success, message }

Transcription:
───────────────

POST /api/transcriptions/upload
  Headers: Authorization: Bearer {token}
  Body: FormData { audio (file), meetingId, language? }
  Response: { transcriptId, text, segmentCount, ... }

GET /api/transcriptions/:meetingId
  Headers: Authorization: Bearer {token}
  Response: { transcript: { segments, content, ... } }

PATCH /api/transcriptions/:transcriptId/segments/:segmentId
  Headers: Authorization: Bearer {token}
  Body: { content?, speakerName? }
  Response: { success, message }

DELETE /api/transcriptions/:transcriptId
  Headers: Authorization: Bearer {token}
  Response: { success, message }

🚨 COMMON ISSUES & SOLUTIONS
================================================================================

Issue: "Cannot find module 'express'"
Solution: 
  cd backend
  npm install

Issue: "PostgreSQL connection refused"
Solution:
  # Check if PostgreSQL is running
  docker ps
  
  # If not running, start it
  docker start smartmeets-db
  
  # Or check local PostgreSQL
  pg_isready -h localhost -p 5432

Issue: "OPENAI_API_KEY is required but not set"
Solution:
  # Check .env file exists
  cd backend
  cat .env | grep OPENAI
  
  # If not set, add it
  echo "OPENAI_API_KEY=sk-xxx" >> .env
  
  # Restart backend
  npm run dev

Issue: "Token invalid" when calling API
Solution:
  # Get new token by logging in
  # Make sure token is in Authorization header: "Bearer token"
  # Check token hasn't expired (default 24h)

Issue: "Audio file too large"
Solution:
  # Max upload is 50MB
  # Convert to MP3 or lower bitrate
  # ffmpeg -i input.wav -b:a 128k output.mp3

Issue: "Port 3000 already in use"
Solution:
  # Kill existing process on port 3000
  lsof -ti:3000 | xargs kill -9
  
  # Or use different port
  PORT=3001 npm run dev

🔄 WORKFLOW EXAMPLE
================================================================================

1. User registers
   └─ POST /api/auth/register
   └─ Token created
   └─ Stored in localStorage

2. User logs in
   └─ POST /api/auth/login
   └─ Token validated
   └─ User profile loaded

3. User starts/attends meeting
   └─ Recording happens
   └─ Audio file created

4. User uploads audio
   └─ POST /api/transcriptions/upload
   └─ OpenAI Whisper processes
   └─ Segments created in DB
   └─ Frontend displays transcript

5. User edits segments
   └─ PATCH /api/transcriptions/{id}/segments/{id}
   └─ Segment updated
   └─ is_edited flag set

6. System generates summary
   └─ Backend calls GPT-4
   └─ Summary + Action Items created
   └─ Frontend displays

7. User exports
   └─ TXT format: Plain text with timestamps
   └─ PDF format: Formatted document
   └─ Both include metadata
